## Goal
На stats-странице рядом с заголовком показать маленький бейдж с общим числом тасков в статусе done.

## Context
- `kanban/static/stats.js` строит страницу через `renderStats()`. Внутри уже формируется `headerHtml`.
- Данные приходят из `/api/stats` (агрегатор). Проверь возвращаемую структуру — там должны быть тотал-счётчики или массив тасков, по которому можно посчитать done.
- Если `/api/stats` не отдаёт нужного — допустимо посчитать на клиенте по `tasksHtml`-источнику данных, но лучше использовать готовое поле, если оно есть.
- Стиль бейджа можно взять подходящий из существующих в styles.css; новые css-классы НЕ добавляй (это сферa T103/T105 при необходимости — здесь только stats.js).

## Acceptance criteria
- [ ] В шапке stats-страницы видно бейдж формата `<N> done`, где N — число завершённых тасков.
  Открой `/stats` (или раздел Stats) — бейдж присутствует и значение совпадает с количеством строк-задач со статусом done в текущем рендере.
- [ ] Бейдж обновляется при ре-рендере страницы (если `renderStats` вызывается повторно — число пересчитывается).
- [ ] Не трогать `styles.css` — только `stats.js`. Если стиль необходим — используй inline-style на элементе.

## How to verify
1. Запусти сервер, открой stats-страницу.
2. Сверь число в бейдже с числом видимых "done" строк.
3. Перейди на другую страницу и обратно — значение перерисовалось.
