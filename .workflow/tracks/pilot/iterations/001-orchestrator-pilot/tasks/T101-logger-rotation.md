## Goal
Добавить в `kanban/lib/logger.mjs` лёгкую ротацию: при первом обращении в новый день удалять директории `logs/YYYY-MM-DD/` старше 14 дней.

## Context
- `kanban/lib/logger.mjs` — текущий код пишет в `logs/YYYY-MM-DD/<source>.log`, без чистки.
- `LOG_BASE` определён там же.
- Кеш `_ready` уже отслеживает `day/source` — можно подвязать чистку к моменту, когда в `_ready` появляется ключ нового дня.

## Acceptance criteria
- [ ] При записи первой строки в новой `YYYY-MM-DD`-папке запускается синхронная (или fire-and-forget) чистка: пройти по `LOG_BASE`, удалить директории, чьё имя — валидная дата старше 14 дней от сегодняшней.
  Создай руками `logs/2026-04-01/` (старше 14 дней) и `logs/2026-04-30/` (свежая), запусти любую запись через logger — старая папка должна исчезнуть, свежая остаться.
- [ ] Чистка выполняется не чаще одного раза за процесс (после первой записи в новый день — больше не повторяется).
- [ ] Все ошибки fs обёрнуты в try/catch; падение чистки не должно ломать запись лога.

## How to verify
1. Создай две тестовые папки `logs/2025-01-01/` и `logs/<сегодня-минус-1>/` с заглушкой `dummy.log`.
2. Запусти `node -e "import('./kanban/lib/logger.mjs').then(m => m.logger.info('test', 'hi'))"` (или через kanban-сервер).
3. Проверь — `logs/2025-01-01/` удалена, свежая папка цела.
